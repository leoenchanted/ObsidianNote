### 通信

RESTful api 是一种基于 HTTP 协议的架构风格，用于设计网络应用程序的 API。

- **无状态性（Stateless）** ：每个请求都包含服务器处理请求所需的全部信息，服务器不会保存客户端的状态。

- **资源导向（Resource-Based）** ：API 的核心是资源（如用户、订单、文章等），每个资源都有一个唯一的标识符（通常是 URL）。

- **统一接口（Uniform Interface）** ：使用标准的 HTTP 方法（GET、POST、PUT、DELETE 等）对资源进行操作。

- **可缓存性（Cacheable）** ：响应可以被缓存以提高性能。

  #### **优点：**

  - **简单易用** ：基于 HTTP 协议，开发者容易理解和实现。
  - **可扩展性强** ：支持缓存、分层系统等特性，适合大规模应用。
  - **标准化** ：使用统一的接口和状态码，便于维护和协作。

  #### **缺点：**

  - **缺乏严格的规范** ：REST 只是一种风格，没有强制性的标准，可能导致不同的实现方式。
  - **不适合复杂场景** ：对于需要频繁交互或实时更新的场景（如聊天应用），REST 可能不够高效。

  频繁的聊天应用用 **WebSocket** 

**WebSocket** 允许客户端和服务器之间建立持久连接，从而实现双向实时通信。
